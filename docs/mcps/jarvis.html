Available Actions

  • update_pipeline
    Re-apply environment & configuration to every package in a Jarvis-CD pipeline
    Parameters: pipeline_id (str)
  • build_pipeline_env
    Rebuild a Jarvis-CD pipeline's env.yaml, capturing only CMAKE_PREFIX_PATH and PATH
    Parameters: pipeline_id (str)
  • create_pipeline
    Create a new Jarvis-CD pipeline environment
    Parameters: pipeline_id (str)
  • load_pipeline
    Load an existing Jarvis-CD pipeline environment
    Parameters: pipeline_id (str, optional)
  • get_pkg_config
    Retrieve the configuration of a specific package in a Jarvis-CD pipeline
    Parameters: pipeline_id (str), pkg_id (str)
  • append_pkg
    Append a package to a Jarvis-CD pipeline
    Parameters: pipeline_id (str), pkg_type (str), pkg_id (str, optional), do_configure (bool, optional), extra_args (dict, optional)
  • configure_pkg
    Configure a package in a Jarvis-CD pipeline
    Parameters: pipeline_id (str), pkg_id (str), extra_args (dict, optional)
  • unlink_pkg
    Unlink a package from a Jarvis-CD pipeline (preserve files)
    Parameters: pipeline_id (str), pkg_id (str)
  • remove_pkg
    Remove a package entirely from a Jarvis-CD pipeline
    Parameters: pipeline_id (str), pkg_id (str)
  • run_pipeline
    Execute a Jarvis-CD pipeline end-to-end
    Parameters: pipeline_id (str)
  • destroy_pipeline
    Destroy a Jarvis-CD pipeline environment and clean up files
    Parameters: pipeline_id (str)
  • jm_create_config
    Initialize JarvisManager config directories
    Parameters: config_path (str, optional)
  • jm_load_config
    Load existing JarvisManager configuration
    Parameters: config_path (str, optional)
  • jm_save_config
    Save current JarvisManager configuration
    Parameters: config_path (str, optional)
  • jm_set_hostfile
    Set hostfile path for JarvisManager
    Parameters: hostfile_path (str)
  • jm_bootstrap_from
    Bootstrap Jarvis config from a machine template
    Parameters: machine_name (str)
  • jm_bootstrap_list
    List available bootstrap machine templates
    Parameters: None
  • jm_reset
    Reset JarvisManager (destroy all pipelines and data)
    Parameters: None
  • jm_list_pipelines
    List all existing Jarvis pipelines
    Parameters: None
  • jm_cd
    Change current Jarvis pipeline context
    Parameters: pipeline_id (str)
  • jm_list_repos
    List all Jarvis repositories
    Parameters: None
  • jm_add_repo
    Add a repository to JarvisManager
    Parameters: repo_name (str), repo_path (str)
  • jm_remove_repo
    Remove a repository from JarvisManager
    Parameters: repo_name (str)
  • jm_promote_repo
    Promote a repository in JarvisManager
    Parameters: repo_name (str)
  • jm_get_repo
    Get repository info from JarvisManager
    Parameters: repo_name (str)
  • jm_construct_pkg
    Construct a package skeleton in JarvisManager
    Parameters: pkg_name (str), pkg_type (str, optional)
  • jm_graph_show
    Print the current resource graph frames
    Parameters: None
  • jm_graph_build
    Build or rebuild the resource graph with a net sleep interval
    Parameters: sleep_interval (float, optional)
  • jm_graph_modify
    Modify the resource graph using a net sleep interval
    Parameters: sleep_interval (float, optional)

Usage Examples

Pipeline Management

// Create a new pipeline create_pipeline(pipeline_id: "my_simulation") // Load existing pipeline load_pipeline(pipeline_id: "my_simulation") // Update pipeline configuration update_pipeline(pipeline_id: "my_simulation") // Build pipeline environment build_pipeline_env(pipeline_id: "my_simulation")

Package Management

// Add package to pipeline append_pkg( pipeline_id: "my_simulation", pkg_type: "application", pkg_id: "openmpi" ) // Configure package configure_pkg( pipeline_id: "my_simulation", pkg_id: "openmpi", extra_args: {"version": "4.1.0"} ) // Get package configuration get_pkg_config(pipeline_id: "my_simulation", pkg_id: "openmpi")

JarvisManager Operations

// Initialize configuration jm_create_config() // List available pipelines jm_list_pipelines() // Add repository jm_add_repo(repo_name: "custom_tools", repo_path: "/path/to/tools") // List repositories jm_list_repos() // Bootstrap from template jm_bootstrap_from(machine_name: "cluster_template")

Resource Graph Management

// Show current resource graph jm_graph_show() // Build resource graph jm_graph_build(sleep_interval: 1.0) // Modify resource graph jm_graph_modify(sleep_interval: 0.5) // Set hostfile for distributed operations jm_set_hostfile(hostfile_path: "/etc/hosts")
