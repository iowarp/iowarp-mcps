Lmod

System Management

Environment module management for scientific computing. Provides dynamic loading and unloading of software modules in HPC environments.

1.0.0 Version
10 Actions
Jan 15 Updated
Python Language

Installation Methods

Global Installation

uv pip install scientific-mcps

Individual MCP Installation

uv pip install "scientific-mcps[lmod]"

Direct Server

uv run mcp_lmod

Claude Desktop Configuration

{ "mcpServers": { "lmod": { "command": "uv", "args": ["run", "mcp_lmod"] } } }

Available Actions

  • module_list
    List all currently loaded environment modules. Shows the active modules in your current shell environment
    Parameters: None
  • module_avail
    Search for available modules that can be loaded. Optionally filter by name pattern (e.g., 'python', 'gcc/*', '*mpi*')
    Parameters: pattern (str, optional)
  • module_show
    Display detailed information about a specific module including its description, dependencies, environment variables it sets, and conflicts
    Parameters: module_name (str)
  • module_load
    Load one or more environment modules into the current session. Modules modify environment variables like PATH, LD_LIBRARY_PATH, etc.
    Parameters: modules (list[str])
  • module_unload
    Unload one or more currently loaded modules from the environment
    Parameters: modules (list[str])
  • module_swap
    Swap one module for another (unload old_module and load new_module atomically)
    Parameters: old_module (str), new_module (str)
  • module_spider
    Search for modules using spider search functionality with detailed dependency information
    Parameters: search_term (str)
  • module_save
    Save the current module environment to a named collection for later restoration
    Parameters: collection_name (str)
  • module_restore
    Restore a previously saved module collection
    Parameters: collection_name (str)
  • module_savelist
    List all saved module collections available for restoration
    Parameters: None

Usage Examples

Basic Module Management

// List currently loaded modules module_list() // Search for available modules module_avail(pattern: "python") module_avail(pattern: "gcc/*") // Get detailed module information module_show(module_name: "python/3.9.0") // Load required modules module_load(modules: ["gcc/11.2.0", "python/3.9.0"])

Advanced Module Operations

// Swap module versions module_swap(old_module: "gcc/10.2.0", new_module: "gcc/11.2.0") // Unload modules module_unload(modules: ["python/3.9.0"]) // Spider search for modules module_spider(search_term: "mpi")

Module Collections

// Save current environment module_save(collection_name: "my_dev_env") // List saved collections module_savelist() // Restore saved environment module_restore(collection_name: "my_dev_env")

HPC Workflow Integration

// Set up environment for job module_load(["gcc/11.2.0", "openmpi/4.1.0", "python/3.9.0"]) // Save this configuration module_save("hpc_simulation") // Submit job with loaded modules submit_slurm_job( script_path: "simulation.sh", cores: 16, memory: "32GB" )
